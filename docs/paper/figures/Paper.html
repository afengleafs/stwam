结论比较直接

  1. STWAM 不是标准 LIBERO SOTA。平均 89.75%，比 2026 公开强 WAM / VLA 低约 7–9 个点。
  2. Object 已接近饱和，98.0% 只比 Fast-WAM / Cosmos / VLA-JEPA 低 0.8–1.6 个点。
  3. 主要差距来自 Spatial、Goal 和 Long，尤其 Long 对 Cosmos / LingBot-VA 差约 15 点。瓶颈不是"识别物体"，而是空间关系、连续交互和关键 fixture affordance。
  4. 相比老 OpenVLA fine-tuned baseline，平均高 13.3 点，Long 高 29.3 点——能说明小型 WAM 结构有效，但不能据此声称超过 2026 SOTA。
  5. 参数 / 算力定位是 STWAM 最有价值的部分。但 Fast-WAM 只训 20k steps，STWAM 训 300k steps，论文里要诚实标注。

Spatial/Object 保持 88%，但 Goal/Long 掉到 44%/41%。模型不是纯记忆所有外观，但对关键交互物、fixture 和长程 affordance 绑定仍依赖训练分布。

✗ Position / Task 主要崩塌

Position 四套全 0%，Task 平均 4.5%。模型对初始空间布局和任务逻辑重定义几乎没有可迁移性。

± Environment 少量迁移

四套平均 29.5%，libero_10 达 43%，但 spatial/object/goal 只有 21–29%。比 Position/Task 好，仍说明场景 / 工作台分布被强记忆。
完整 LIBERO-PRO 扰动均值为 37.65%——五类 × 四套共 20 个 cell 的平均值。它应作为鲁棒性弱点而非卖点来呈现。

直接比较

  1. Semantic 89.0% 接近 π0 的 90.5%，高于 MolmoAct/NORA，低于 OpenVLA/π0.5/x-VLA。语言扰动鲁棒性是可写进论文的正结果。

05对 STWAM 论文定位的建议写法

STWAM demonstrates a parameter-efficient semantic-latent world-action model: it combines Fast-WAM-style training-time video co-training and inference-time direct action prediction with frozen V-JEPA semantic latents, achieving 89.75% average success on standard LIBERO using a 0.44B-parameter model and no embodied pretraining.

可以强调的创新

  • 语义 latent WAM 闭环。把 Fast-WAM 的 video co-training 与 VLA-JEPA 的 JEPA 表征合到一个小型 WAM。
  • 参数效率。相比 5B–8B 的 WAM/VLA，总体约 0.44B、可训约 0.10B，冻结大视觉编码器。
  • 语言扰动鲁棒。LIBERO-PRO Semantic 四套平均 89.0%，几乎保持原始成功率。
  • 鲁棒性诊断清楚。失败集中在 Position、Task、Environment，改进目标明确。

必须避免的过强表述

  • 不要说达到 2026 SOTA——标准 LIBERO 比 Fast-WAM/Cosmos/LingBot-VA 低 7–9 点。
  • 不要笼统说 "object generalization 很强"——Goal/Object、Long/Object 掉到 44%/41%。
  • 不要说 LIBERO-PRO 泛化强——完整均值仅 37.65%，Position 0%，Task 4.5%。
  • 不要把 Spatial/Position 弱点藏起来——这恰是后续工作的最清晰方向。

最应该补的实验

  1. Spatial/Position 诊断：增加 object pose / slot / depth / multi-frame history 的消融。
  2. Task 泛化诊断：分析 task perturbation 中语言目标与 BDDL target-state 的变化是否被策略忽略。
  3. 语义 latent vs 像素/VAE latent 的受控消融，对齐 Fast-WAM 的 "video co-training" ablation。
  4. 训练步数与模型尺寸消融：100k/200k/300k，semantic_dim 96 vs 更高维，num_history 1/2/4。
